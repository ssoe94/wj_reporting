# Production Plan Upload & Daily Dashboard Design

## 목적

생산 계획 엑셀을 기준 데이터로 삼아, 사출과 가공 담당자가 같은 날짜의 작업 계획을 빠르게 확인하고 생산 결과 기록으로 이어지게 한다.

- 한국어/중국어 병용 UI를 기본으로 한다.
- 엑셀 파싱은 기존 백엔드 `ProductionPlanUploadView`와 `ProductionPlanProcessor`를 우선 재사용한다.
- 새 프런트는 업로드, 파싱 결과 확인, 일자별 대시보드, 상세 행 조회까지를 하나의 운영 흐름으로 묶는다.

## 현재 API 계약

### Upload

`POST /production/plan/upload/`

`multipart/form-data`

| 필드 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `file` | File | Y | `.xlsx`, `.xls` 생산계획 파일 |
| `plan_type` | string | Y | `injection` 또는 `machining` |
| `date` | string | Y | 업로드 기준일 `YYYY-MM-DD` |

응답은 백엔드 파서가 만든 원본 요약을 그대로 반환한다.

| 필드 | 설명 |
| --- | --- |
| `plan_type` | 파싱된 계획 유형 |
| `available_days` | 실제 저장 대상 날짜 목록 |
| `plan_long` | 일자별로 펼쳐진 상세 계획 행 |
| `records` | 원본 기준 행 |
| `machine_summary` | 설비별 계획 수량 요약 |
| `model_summary` | 모델별 계획 수량 요약 |

### Summary

`GET /production/plan-summary/?date=YYYY-MM-DD`

사출과 가공을 같은 날짜 기준으로 함께 반환한다.

### Detail Rows

`GET /production/plans/?date=YYYY-MM-DD&plan_type=injection|machining`

선택 날짜와 공정의 상세 계획 행을 반환한다.

## 화면 구성

### 상단 기준값

- `계획일 / 计划日期`
- `공정 / 工序`: 사출/가공 전환
- `언어 / 语言`: 한국어/중국어 전환, 새로고침 후 유지

### 일자별 KPI

- 계획 수량: 선택 날짜의 사출+가공 합계
- 설비/라인: 계획이 배정된 설비와 가공 라인 수
- 작업 행: 저장된 상세 계획 행 수

### 업로드 패널

- 업로드 기준일
- 공정 선택
- 엑셀 파일 선택
- 업로드 및 파싱 실행
- 성공/실패 메시지 표시
- 성공 시 해당 날짜와 공정으로 화면 이동 후 요약/상세 재조회

### 일자별 대시보드

- 같은 날짜에서 사출과 가공의 계획 수량을 나란히 비교
- 각 공정 카드 클릭 시 상세 목록과 설비/모델 요약이 해당 공정으로 전환
- MVP에서는 계획 수량 중심으로 표시하고, 다음 단계에서 실적/불량/정지 이슈를 연결한다.

## 파싱 기준

기존 파서는 다음 구조를 기대한다.

- 시트명은 업로드 기준일의 월-일 형식이다. 예: `5-9`, `5-09`
- 헤더는 3번째 행 기준으로 읽는다.
- 사출과 가공은 서로 다른 열 매핑을 사용한다.
- 숫자형 날짜 열을 일자별 계획 수량으로 펼친다.
- 업로드 시 같은 날짜 범위와 같은 `plan_type`의 기존 계획은 삭제 후 재생성한다.

## 다음 구현 순서

1. 실제 회사 엑셀 샘플 1개씩으로 파싱 성공/실패 케이스를 확인한다.
2. 업로드 응답에 `created_count`, `deleted_count`, `warning_rows`를 추가해 운영자가 결과를 검증할 수 있게 한다.
3. 상세 목록에서 설비별 순서, Lot, Part No, 계획수량을 편집 없이 확인하는 읽기 화면을 먼저 안정화한다.
4. 생산 실행 화면에서 계획 행을 선택해 불량률, 생산 현황, 이슈 기록으로 이어지게 한다.
5. 사출 모니터링 데이터와 연결해 시간당 생산량 변화, 추정 CT, 급감/정지 이슈 입력을 붙인다.
